# 一、面向对象编程
1. OOP思想
	+ go语言不是面向对象的语言，只是让大家理解一些面向对象的思想，通过一些方法来模拟面向对象
	+ 核心：封装、继承、多态

2. 继承
	+ 继承需要符合的关系是：is-a，父类更通用，子类更具体
	+ 继承就是子类继承父类的特征和行为，使得子类具有父类的属性和方法，使得子类具有父类相同的行为。子类会具有父类的一般特性也会具有自身的特性
	+ go语言中结构体嵌套
		1. 模拟继承:is-a
		```
			type A struct{
				int_a int
				...
			}
			type B struct{
				A		//匿名字段 B.A
				...
			}
			// B是可以直接访问A的属性的：
			b := B{}
			b.int_a = 10
			fmt.Println(b.int_a)	//10
		```
		2. 聚合关系:has-a
		```
			type C struct{
				int_c int
				...
			}
			type D struct{
				c C		//聚合
				...
			}
			// D是无法直接访问C中的属性的，必须使用D.c.int_c
		```
